## Status Report Template

Used by: [status.md](status.md)

```markdown
## E2E Reverse Engineering Status

**Session**: {status} (iteration {iteration}/{max_iterations})
**Started**: {started_at}

### Coverage

- Pages discovered: {pages_discovered}
- Pages documented: {pages_documented} ({percentage}%)
- Scenarios total: {scenarios_total}
- Average quality: {avg_quality_score.toFixed(2)} / 1.0

### Quality Distribution

| Quality Range | Pages | Percentage |
|---------------|-------|------------|
| 0.85+ (Excellent) | {excellent} | {excellent_pct}% |
| 0.70-0.84 (Good) | {good} | {good_pct}% |
| 0.50-0.69 (Fair) | {fair} | {fair_pct}% |
| <0.50 (Needs Work) | {needs_work} | {needs_work_pct}% |

### Page Status

| Page | Priority | Quality | Visits | Scenarios | Coverage Gaps |
|------|----------|---------|--------|-----------|---------------|
{for each page in visit_history:}
| {page.path} | {page.priority} | {page.quality_score.toFixed(2)} | {page.visit_count} | {page.scenarios.length} | {page.coverage.states_missing.join(', ') || 'none'} |
{end for}

### Next Actions

{if pages_to_revisit.length > 0:}
**Pages needing attention** (sorted by urgency):
{for each page in pages_to_revisit.slice(0, 5):}
- {page.path}: {page.reason} (quality: {page.quality_score.toFixed(2)})
{end for}
{else:}
All pages meet quality targets! 🎉
{end if}

### Feature Files

{for each feature in features:}
- [{feature.name}]({feature.path}): {feature.scenario_count} scenarios, quality {feature.quality_score.toFixed(2)}
{end for}

**Total features**: {features.length}
```

## Export Report Templates

Used by: [export.md](export.md)

### Markdown Format

`.claude/e2e-reverse-report.md`:

```markdown
# E2E Reverse Engineering Report

**Generated**: {timestamp}
**Session Status**: {status}
**Iterations**: {iteration}/{max_iterations}

## Summary

- **Total Pages Discovered**: {pages_discovered}
- **Pages Documented**: {pages_documented} ({percentage}%)
- **Total Scenarios**: {scenarios_total}
- **Average Quality Score**: {avg_quality_score.toFixed(2)} / 1.0

## Quality Breakdown

| Quality Range | Pages | Percentage |
|---------------|-------|------------|
| 0.85+ (Excellent) | {excellent} | {excellent_pct}% |
| 0.70-0.84 (Good) | {good} | {good_pct}% |
| 0.50-0.69 (Fair) | {fair} | {fair_pct}% |
| <0.50 (Needs Work) | {needs_work} | {needs_work_pct}% |

## Coverage Detail

| Page | Priority | Quality | Visits | Scenarios | Coverage Gaps |
|------|----------|---------|--------|-----------|---------------|
{for each page in visit_history:}
| {page.path} | {page.priority} | {page.quality_score.toFixed(2)} | {page.visit_count} | {page.scenarios.length} | {gaps} |
{end for}

## Recommendations

{if needs_work_pages.length > 0:}
### Pages Needing Improvement

{for each page in needs_work_pages:}
- **{page.path}** (quality: {page.quality_score.toFixed(2)})
  - Missing states: {page.coverage.states_missing.join(', ')}
  - Missing devices: {page.coverage.devices_missing.join(', ')}
  - Missing roles: {page.coverage.roles_missing.join(', ')}
  - Recommendation: {page.recommendation}
{end for}
{end if}

### Coverage Gaps

{for each gap_type in ['error states', 'empty states', 'mobile coverage']:}
- {pages_missing[gap_type].length} pages missing {gap_type}
{end for}

## Feature Files

{for each feature in features:}
- **[{feature.name}]({feature.path})**
  - Scenarios: {feature.scenario_count}
  - Quality: {feature.quality_score?.toFixed(2) || 'N/A'}
  - Tags: {feature.tags.join(', ')}
{end for}

## Configuration

- **Base URL**: {config.base_url}
- **Devices**: {config.devices.map(d => d.name).join(', ')}
- **Language**: {config.language}
- **Output Directory**: {config.output_dir}
- **Screenshot Directory**: {config.screenshot_dir}

---

*Generated by E2E Reverse Engineering on {timestamp}*
```

## Validation Report Template

Used by: [validate.md](validate.md)

`.claude/e2e-reverse-validation.md`:

```markdown
# Gherkin Validation Report

**Validated**: {timestamp}
**Files Checked**: {file_count}

## Summary

- ✅ Valid: {valid_count} files
- ⚠️  Warnings: {warning_count} files
- ❌ Errors: {error_count} files

## Issues Found

{for each file in files_with_issues:}
### {file.name} {status_icon}

{if file.errors.length > 0:}
**Errors** ({file.errors.length}):
{for each error in file.errors:}
- **Line {error.line}**: {error.message}
  - {error.suggestion}
{end for}
{end if}

{if file.warnings.length > 0:}
**Warnings** ({file.warnings.length}):
{for each warning in file.warnings:}
- **Line {warning.line || 'N/A'}**: {warning.message}
  - {warning.suggestion}
{end for}
{end if}

{if file.coverage_gaps.length > 0:}
**Coverage gaps**:
{for each gap in file.coverage_gaps:}
- {gap.type}: {gap.description}
{end for}
{end if}

{end for}

## Recommendations

{if error_count > 0:}
### ⚠️ Fix Syntax Errors Before Running Tests

{error_count} files have syntax errors that must be fixed:
{for each file in files_with_errors:}
- {file.name}: {file.errors.length} errors
{end for}
{end if}

{if warning_count > 0:}
### 💡 Improve Coverage and Quality

{warning_count} files have warnings:
{for each warning_type in aggregated_warnings:}
- {warning_type.count} files missing {warning_type.type}
{end for}
{end if}

{if all_valid:}
### ✅ All Files Valid!

All {file_count} feature files pass validation. Great work!
{end if}

## Coverage Analysis

Features meeting quality target ({target_quality_score}): {high_quality_count} / {file_count} ({percentage}%)

{if low_coverage_features.length > 0:}
### Features Needing Attention

{for each feature in low_coverage_features:}
- **{feature.name}** (quality: {feature.quality_score.toFixed(2)})
  - Missing: {feature.gaps.join(', ')}
  - Recommendation: {feature.recommendation}
{end for}
{end if}

---

*Run validation again with `/e2e-reverse validate` after making changes*
```

## Usage Guidelines

### For Command Developers

When implementing status/export/validate/checkpoint commands:

1. **Load data** directly:
   - Read `.claude/ralph-loop.local.md` for session state
   - Read `.claude/e2e-reverse.config.md` for configuration
   - Use `grep -c "Scenario:"` for scenario counts
   - Use `Glob` to list feature files

2. **Select template** from this file based on command

3. **Populate variables** with actual data

4. **Render template** and write to appropriate location

### Template Variables

Common variables used across templates:

- `{timestamp}`: Current ISO8601 timestamp
- `{status}`: Session status (running/paused/completed)
- `{iteration}`, `{max_iterations}`: Progress tracking
- `{pages_discovered}`, `{pages_documented}`: Page counts
- `{scenarios_total}`: Total scenario count
- `{avg_quality_score}`: Average quality (0-1)
- `{excellent}`, `{good}`, `{fair}`, `{needs_work}`: Quality distribution counts
- `{visit_history}`: Array of page objects
- `{features}`: Array of feature file objects
- `{config}`: Configuration object

### Formatting Helpers

**Percentage**:
```javascript
const percentage = Math.round((pages_documented / pages_discovered) * 100)
```

**Quality Class** (for HTML):
```javascript
const qualityClass =
  quality >= 0.85 ? 'excellent' :
  quality >= 0.70 ? 'good' :
  quality >= 0.50 ? 'fair' : 'needs-work'
```

**Status Icon**:
```javascript
const statusIcon =
  errors > 0 ? '❌' :
  warnings > 0 ? '⚠️' : '✅'
```
